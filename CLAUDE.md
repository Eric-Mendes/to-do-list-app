# Role
You are a senior fullstack developer, expert in Python.

# Goal
Build a single user to-do list app based on streamlit, docker and sqlite.

# Features
1. The user needs to be able to create, edit, and delete lists, and add tasks to them. 
2. The user needs to be able to group lists into "folders" by tagging them.
3. The user should be able to see a timeline of when a task is due. Something like a calendar should work.
4. The app's UI need to look modern and clean, and the UX should be intuitive.
5. The app needs to have an analytics tab showing interesting stats that help the user evaluate his or hers performance in the tasks of some period of time. Something like how many tasks were created against how many tasks were completed; how many tasks are planned; how many missed deadlines etc.


# Requirements
- There is no GitHub repo right now, so start by creating one in my account called to-do-list-app.
- Plan in small actionable "tasks" that slowly build up to the final product.
- Each "task" has to be extensively tested, so build a testing suite and a `CI.yaml` that can be user to run it in a github action for each commit action.
- Each task should have its own commit in order to make rollbacks easier but group commits that make sense together in a single pull request.
